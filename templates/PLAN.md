# PLAN

Short index of work plans. Link detailed plans under `docs/plan/`.

## Planning Rules

- Detect repository stack and required LSP/check tools before implementation.
- Install or bootstrap missing required tools before declaring them unavailable.
- Record tool readiness, dry-run/check evidence, module acceptance evidence, and surface integration evidence in detailed plans.

## Active Plans

- `docs/plan/<topic>.md`: short status.

## Completed Plans

- `docs/plan/<topic>.md`: evidence summary.
